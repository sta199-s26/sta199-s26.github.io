Two important concepts

Computational reproducibility:

  • If you give me your code and your data and I hit ‚ÄúRun,‚Äù do I get the same numbers and the same pictures that you published in your article?
  • If so‚Ä¶phew!
  • If not‚Ä¶what the ü§¨?

Scientific replication:

  • Can new researchers conduct an independent analysis and reach the same substantive conclusions that you did?
  • Just because your analysis is computationally reproducible does not mean that you have a robust result that generalizes.

Our tools will help you achieve the first, which is necessary (but not sufficient!) for the second.

Reproducibility checklist

What does it mean for a data analysis to be ‚Äúreproducible‚Äù?

Short-term goals:

  • Are the tables and figures reproducible from the code and data?
  • Does the code actually do what you think it does?
  • In addition to what was done, is it clear why it was done?

Long-term goals:

  • Can the code be used for other data?
  • Can you extend the code to do other things?

Toolkit for reproducibility

  • Scriptability ‚Äì Coding instead of point-and-click \(\rightarrow\) R
  • Literate programming ‚Äì code, written text, results all in one place instead of error-prone copy-paste \(\rightarrow\) Quarto
  • Version control ‚Äì Track changes with brief messages that describe those changes \(\rightarrow\) Git / GitHub
